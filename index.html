CASE 01 SCORM 1.2 Go1 8Squad

Delivery KPIs & Performance Review Framework

Interactive enablement microsite · for delivery consultants
index.html — your performance, your roadmap

A dense HR framework — four KPI areas, a 1–5 scoring scale, a review process and a spreadsheet template — rebuilt as a single-scroll, chaptered course that a consultant can finish in about ten minutes and actually remember. The brief was enablement, not a document dump.

  • Live score simulator — sliders for each KPI feed a weighted overall score in real time, so people see how the maths works on their own numbers.
  • Interactive KPI tabs with a proportional weighting bar — measures, behaviours and risks per KPI.
  • Five-step review-form walkthrough with a mock spreadsheet preview of the real template.
  • Knowledge check — five questions with inline, explain-why feedback that ties back to the "why this exists" section.
  • Resolved a source-of-truth conflict — the KPI weightings differed between the DOCX and the PPTX; I surfaced it, built a toggle, and flagged it for sign-off before rollout.
  • Voice tuned to context — reworked the headline away from a "we're scaling" framing to "Clear measures. Project-by-project evidence." after a round of redundancies.

CASE 02 SCORM 1.2 Go1 Employment Hero 8Squad

Psychosocial Hazards Learning Path

Curated WHS learning path · leaders & all staff · VIC · NSW · QLD
index.html — managing psychosocial hazards, together

Newly regulated territory across three states, and no single course that covered it. I built a curated learning path that threads government training, the state codes of practice, peer-support resources and our own behavioural material into one guided, trackable journey — brand-safe and mostly free to run.

  • Four-tier structure — core courses, codes of practice, peer support, and a behavioural layer, each tagged by audience and resource type.
  • Eleven curated resources with a clear tag taxonomy (SCORM course / watch here / new tab / PDF / internal) so learners always know what they're clicking.
  • Sequencing guide — a recommended rollout order for anyone running the path across a business unit.
  • Knowledge check with an 80% mastery gate — completion is only reported to the LMS once the learner passes.
  • Resume where you left off — progress and quiz state persist across sessions via SCORM suspend data.
  • Compliance-mapped — Queensland set as the compliance "ceiling" so building to it satisfies every state we operate in.

8Squad Design System

The visual language behind the courses — extracted & documented

So the courses would feel like one product, I distilled the work into a documented design system: colour tokens, a two-family type scale, and a component library that every module is built from.

  • Tokenised colour — a Bondi Blue primary family plus semantic tokens that keep light and dark modes in sync.
  • Component library — buttons, badges, cards, forms, rating scales, weight bars and result panels, each with usage rules.
  • Type system — Roboto for interface, Roboto Mono for data and labels; a full display-to-caption scale.

04 · Under the hood

The SCORM craft, not just the slides

These packages are hand-authored down to the manifest. That's what makes them small, dependency-free, and reliable to upload — and it's why a rejected package can be diagnosed and fixed rather than re-exported and hoped over.

01

Valid, minimal manifest

A clean imsmanifest.xml at the package root — correct namespaces, a single sco resource, every dependency declared. No wrapping folder, no orphaned references.

02

Completion & mastery

Progress and results written through the LMS runtime API — lesson_status, score.raw and a masteryscore that matches the quiz's pass mark, so completion reports the way the LMS expects.

03

Resume with suspend data

Learner state — which resources are done, the quiz outcome — is serialised into cmi.suspend_data and restored on return, so a two-hour path survives being closed.

04

Self-contained & debuggable

No build step and no framework CDN in the shipped package — it runs offline inside the LMS iframe, and when a platform like Go1 is fussy, the package can actually be inspected and corrected.
